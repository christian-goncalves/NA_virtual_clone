# File Priority Order

## Prioridade de leitura obrigatoria

1. `kb/`
2. `standards/`
3. `patterns/`
4. `templates/`
5. codigo real do projeto

## Regra operacional

Sempre que houver duvida de implementacao, revisar esta ordem e validar aderencia aos documentos antes de concluir geracao ou revisao de codigo.

## Justificativa

- `kb/` define arquitetura e intencao tecnica
- `standards/` define governanca e limites
- `patterns/` transforma principios em implementacao pratica
- `templates/` acelera consistencia na criacao de arquivos
- codigo real e usado para adaptar contexto sem perder padrao
